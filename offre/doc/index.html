Mission #0

Éditeur Markdown in-browser · Single-file HTML · Civic tech tool
HTML5 Vanilla JS Marked.js Highlight.js Mermaid.js ouaisfieu.github.io

Mission #0 est un éditeur Markdown entièrement autonome fonctionnant dans le navigateur. Conçu pour l'écosystème ouaisfi.eu, il combine un arbre de fichiers, un éditeur live, un aperçu en temps réel et des exports MD/HTML — le tout en un seul fichier HTML déployable sans backend.

Stack technique

L'application ne dépend d'aucun framework frontend. Elle repose sur trois bibliothèques chargées via CDN local (fichiers *.js.téléchargement inclus dans le dossier d'assets) :

BibliothèqueRôleConfig notable
marked.min.js Parsing Markdown → HTML gfm: true, breaks: true, headerIds: true
highlight.min.js Coloration syntaxique des blocs de code Thème Tokyo Night Dark (tokyo-night-dark.min.css)
mermaid.min.js Rendu de diagrammes depuis blocs ```mermaid Thème custom lilac/vert terminal, startOnLoad: true
⟩ Note de déploiement
Les scripts sont référencés en chemins relatifs depuis ./Mission #0_files/. L'application fonctionne offline dès lors que ce dossier d'assets est présent à côté du fichier HTML.

Architecture

L'application est structurée en un unique fichier HTML avec trois zones distinctes :

[ HTML + CSS ] → Structure visuelle complète (inline, ~600 lignes) │ ├─ [ <body> statique ] → UI pré-rendue avec état initial (guide.md affiché) │ Le preview et l'arbre fichiers sont hydratés côté JS │ └─ [ <script> JS ] → Logique applicative (~540 lignes) Initialisation, gestion état, fetch, events, exports Flux de données : Fichiers .md (fetch async) ──→ files[] (tableau en mémoire) ──→ updateFileTree() → DOM sidebar ──→ openFile(index) → editor.value ──→ updatePreview() → marked.parse() → preview.innerHTML ──→ updateStats() → compteurs DOM

État global

L'application maintient trois variables globales :

VariableTypeDescription

Structure d'un objet FileObj

{
  name:        String,    // nom du fichier (ex: "guide.md")
  content:     String,    // contenu Markdown brut
  path:        String,    // chemin relatif (uniquement si fromFolder: true)
  fromFolder:  Boolean,   // true = chargé via fetch, false = uploadé par l'utilisateur
  modified:    Boolean    // true si modifications non sauvegardées
}

Panneau fichiers

La sidebar gauche (fond vert terminal, box-shadow: 0 0 30px rgba(0,255,65,0.2)) contient trois éléments fonctionnels :

Zone d'upload

Label .upload-area lié à un <input type="file" accept=".md,.markdown" multiple>. Supporte le glisser-déposer (drag & drop) via les événements dragenter, dragover, dragleave, drop. Seuls les fichiers .md et .markdown sont acceptés — les autres déclenchent un toast d'erreur.

Arbre de fichiers

Rendu dynamique via updateFileTree() à partir du tableau files[]. Chaque item est un .file-item cliquable appelant openFile(index). L'item actif reçoit la classe .active (bordure verte, fond vert translucide).

⟩ Distinction visuelle
Les fichiers chargés depuis le dossier serveur affichent l'icône 📄 (fromFolder: true), les fichiers uploadés manuellement affichent 📝 (fromFolder: false).

Éditeur

L'éditeur est un simple <textarea#editor> en police monospace. Il écoute l'événement input pour déclencher en temps réel :

  • La mise à jour de l'aperçu (updatePreview())
  • La mise à jour des statistiques (updateStats())
  • Le marquage du fichier comme modifié (currentFile.modified = true)

Fonction insertText(before, after)

function insertText(before, after = '') {
  const start = editor.selectionStart;
  const end   = editor.selectionEnd;
  const text  = editor.value;
  const selectedText = text.substring(start, end);

  editor.value = text.substring(0, start)
               + before + selectedText + after
               + text.substring(end);

  // Replace cursor after inserted text
  const newPos = start + before.length + selectedText.length;
  editor.setSelectionRange(newPos, newPos);
  editor.focus();

  updatePreview();
  updateStats();
  if (currentFile) { currentFile.content = editor.value; currentFile.modified = true; }
}

Si du texte est sélectionné, il est encadré. Sinon, le curseur se place après before.

Aperçu en temps réel

Rendu Markdown

La fonction updatePreview() appelle marked.parse(editor.value) puis injecte le HTML résultant dans #preview via innerHTML. Elle effectue ensuite une substitution regex pour transformer les blocs ```mermaid en <div class="mermaid"> avant d'appeler mermaid.run({ querySelector: '.mermaid' }).

function updatePreview() {
  let html = marked.parse(editor.value);

  // Transform mermaid code blocks into mermaid divs
  html = html.replace(
    /<pre><code class="language-mermaid">([\s\S]*?)<\/code><\/pre>/g,
    '<div class="mermaid">$1</div>'
  );

  preview.innerHTML = html;
  mermaid.run({ querySelector: '.mermaid' });
}

Gestion des fichiers

Chargement initial — loadFolder()

Au démarrage, loadFolder() itère sur la liste statique markdownFiles[] et effectue un fetch(MARKDOWN_FOLDER + filename) pour chaque entrée. Les fichiers répondant avec response.ok sont poussés dans files[] avec fromFolder: true. En cas d'échec (404, réseau), l'erreur est silencieuse (console.log uniquement). Une fois le chargement terminé, updateFileTree() et openFile(0) sont appelés, et un toast confirme le nombre de fichiers chargés.

⟩ Attention
Le chemin de base est ./markdown/ (constante MARKDOWN_FOLDER). L'application doit donc être servie depuis un contexte HTTP — elle ne fonctionnera pas en file:// pour cette fonctionnalité (CORS/fetch restrictions).

Upload manuel — handleFiles()

Déclenché par le sélecteur de fichiers ou le drag & drop. Appelle loadUploadedFile(file) pour chaque fichier valide. Un FileReader lit le contenu texte et l'ajoute à files[]. Si un fichier de même nom existe déjà, il est remplacé (files[existingIndex] = fileData).

Nouveau fichier — newFile()

Crée un objet FileObj avec le nom nouveau-{Date.now()}.md, le contenu par défaut # Nouveau document\n\nCommencez à écrire ici..., fromFolder: false et modified: true. L'ouvre immédiatement.

Avertissement de quitte

L'événement beforeunload vérifie si files.some(f => f.modified). Si oui, il appelle e.preventDefault() et assigne e.returnValue = '' pour afficher la boîte de dialogue native du navigateur.

Fonctions d'export

Les trois fonctions d'export partagent le même mécanisme de téléchargement :

const blob = new Blob([content], { type: 'text/markdown' });
const url  = URL.createObjectURL(blob);
const a    = document.createElement('a');
a.href     = url;
a.download = filename;
a.click();
URL.revokeObjectURL(url);

exportMd()

Export brut du contenu editor.value en fichier .md. Utilise le nom depuis #filenameInput ou le nom du fichier courant, avec fallback sur 'document.md'.

saveToFolder()

Identique à exportMd() mais met également à jour currentFile.content, currentFile.name et passe currentFile.modified à false. Rafraîchit l'arbre de fichiers.

exportHtml()

Génère un fichier HTML standalone complet incluant :

  • Le contenu preview.innerHTML (déjà converti en HTML)
  • Une feuille de style inline complète (thème lilac/vert terminal)
  • Support tables, blockquote, code, images, liens

Le nom du fichier est dérivé du nom .md avec substitution de l'extension (.replace('.md', '.html')).

Limites connues

⟩ Persistence
Aucune persistance localStorage ou IndexedDB. Tout le contenu réside en RAM. Un rechargement de page efface les fichiers uploadés manuellement et les modifications non exportées.
⟩ Fetch CORS
Le chargement des fichiers depuis ./markdown/ nécessite un contexte HTTP servi. En file://, les fetch échoueront silencieusement et aucun fichier ne sera pré-chargé.
⟩ Sauvegarde
"Sauvegarder" déclenche un téléchargement navigateur — il n'y a pas d'écriture directe sur le filesystem. L'utilisateur doit remplacer manuellement les fichiers sources si besoin.
⟩ Vue Preview — bouton App
Le bouton 📱 App est un lien externe habillé en bouton de toolbar. Il ne bascule pas l'état currentView et ne désactive pas les autres boutons de vue.
