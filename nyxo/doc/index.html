NYXO — Documentation Technique

Documentation complète de l'Agence de Renseignement Citoyenne : une infrastructure numérique pour la santé mentale, la cartographie des ressources, et l'éducation populaire à Bruxelles.

📖 Vue d'ensemble

NYXO est une Progressive Web App (PWA) single-file en HTML/CSS/JS pur, conçue comme une infrastructure citoyenne pour la santé mentale à Bruxelles. Elle centralise les ressources du secteur SSM (Soins de Santé Mentale), gamifie l'engagement citoyen, et fonctionne hors-ligne.

Le nom « Agence de Renseignement Citoyenne » place l'utilisateur dans une posture d'agent actif — non pas spectateur du système de soin, mais acteur informé capable de cartographier, relier et mobiliser les ressources de son territoire.

🎯 Mission & Philosophie

NYXO s'inscrit dans la démarche Voir / Juger / Agir (VJA), méthodologie d'éducation permanente qui guide l'usager de l'observation des faits à l'action collective, en passant par l'analyse critique.

🧭
Bruxelles pilote, monde suivra.
Le projet est explicitement conçu comme un modèle reproductible : une ville-laboratoire dont l'infrastructure peut être dupliquée dans d'autres contextes urbains.

🏗️ Architecture Technique

L'application est un fichier HTML unique auto-contenu (~10 000 lignes), sans dépendance de build, sans serveur back-end requis. Elle charge deux bibliothèques externes via CDN : Leaflet.js (cartographie) et SheetJS/XLSX (export).

Système de navigation

La navigation est gérée par un système de modules CSS (.module / .module.active) : chaque section de l'application est un bloc <section class="module"> qui est affiché ou masqué via JavaScript en fonction du clic dans la sidebar, sans rechargement de page.

JS Système de navigation par modules 
// Activation d'un module
function navigate(moduleId) {
  document.querySelectorAll('.module').forEach(m => {
    m.classList.remove('active');
  });
  document.getElementById(`mod-${moduleId}`).classList.add('active');

  document.querySelectorAll('.nav-item').forEach(n => {
    n.classList.remove('active');
  });
  document.querySelector(`[data-module="${moduleId}"]`)
    ?.classList.add('active');
}

🏠 Lieux de Liens

Répertoire de 20 espaces communautaires bruxellois : maisons de quartier, centres culturels, espaces de rencontre. Chaque fiche comprend adresse, commune, horaires, contact, et tags thématiques.

Les fiches sont filtrables par commune et par type. Un panneau latéral glissant (.detail-panel) affiche le détail complet sans quitter le contexte de la liste.

🏥 SSM & Services

11 Services de Santé Mentale référencés, structurés selon le standard FHIR R4 HealthcareService. Chaque fiche inclut le réseau 107 de rattachement, les spécialités, les modalités d'accès et les coordonnées.

⚠️
Les données SSM sont statiques dans la version actuelle. Une intégration avec une API FHIR live (ex. registre national des prestataires de soins) est prévue dans les versions futures.

🔗 Réseaux 107

Les 4 réseaux de concertation issus de la réforme "107" de la psychiatrie belge (2010) sont documentés : chaque réseau regroupe les acteurs SSM d'un territoire, favorise les soins dans le milieu de vie et réduit l'hospitalisation.

📝 Journal de Bord

Carnet de notes personnel de l'agent. Chaque entrée est horodatée et stockée en localStorage. Le journal peut être exporté en JSON ou TXT. Les entrées sont également générées automatiquement par certaines actions système (ex : activation des notifications push, synchronisation).

📱 PWA & Mode Hors-ligne

NYXO implémente le protocole complet Progressive Web App avec Service Worker, permettant un fonctionnement intégral sans connexion internet après le premier chargement.

Capacités PWA

CapacitéImplémentationStatut
InstallationWeb App Manifest (manifest.json), prompt natif✅ Actif
OfflineService Worker + Cache API✅ Actif
Sync différéeBackground Sync via MessageChannel✅ Actif
Notifications PushPushManager + VAPID✅ Configurable
Indicateur offlineBannière animée en haut d'écran✅ Actif
Taille cacheAffichée dans les Paramètres✅ Actif
📡
Quand l'application passe hors-ligne, une bannière gradient rouge/orange apparaît en haut. Lors de la synchronisation, elle vire au bleu/vert (.offline-indicator.syncing).

FormatDirectionContenu
